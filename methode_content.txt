Guide Procedural ÔÇö Deploiement Web via IA
1. Introduction
Ce guide formalise le processus complet de creation et de deploiement de sites web statiques optimises
pour le SEO local, en utilisant Claude (Anthropic) comme assistant central de production. L'objectif est
l'acquisition de leads qualifies via des requetes ultra longue traine geolocalises.
Cas d'usage type : creation d'un site vitrine pour un prestataire de services (ex. : eradication de
punaises de lit) avec une page dediee par ville d'un departement, chaque page ciblant des requetes SEO
locales specifiques.
Philosophie du workflow : separer la reflexion strategique (Claude Web) de l'execution technique
(Claude Code via VS Code), puis deployer rapidement via un pipeline GitHub > Netlify.
2. Ecosysteme technique : les 4 plateformes
Le workflow repose sur quatre outils complementaires, chacun intervenant a un moment precis du
processus.

5. Points de vigilance
Liste des erreurs frequentes et des pieges a eviter lors de l'utilisation de Claude pour la generation de
sites web.
5.1 Patterns IA a surveiller
ATTENTION : Defauts recurrents de Claude
Claude a tendance a introduire des patterns reconnaissables qu'il faut systematiquement corriger. Verifier apres
chaque generation : les accents manquants ou incorrects, les tirets cadratin utilises a la place de tirets simples,
les majuscules abusives dans les titres Hn (seule la premiere lettre doit etre en majuscule en francais).
ÔÇó Accents : Verifier que tous les accents sont presents et corrects (e, e, a, u, c). Claude oublie
parfois des accents.
ÔÇó Tirets : Remplacer les tirets cadratin (ÔÇö) par des tirets normaux (-) dans les contenus web.
ÔÇó Majuscules Hn : En francais, seul le premier mot d'un titre prend une majuscule. Corriger les
titres type 'Nos Services De Desinsectisation'.
ÔÇó Formulations generiques : Demander a Claude de varier les tournures et d'eviter les phrases
types IA comme 'Dans un monde ou...' ou 'Il est important de noter que...'.
5.2 Erreurs techniques courantes
ÔÇó Formulaires Netlify : Chaque formulaire doit avoir l'attribut netlify et un name unique. Sans cela,
Netlify ne captera pas les soumissions.
ÔÇó Canonical : Chaque page doit avoir une balise <link rel='canonical'> pointant vers sa propre URL
canonique. Oubli frequent.
ÔÇó Redirections : Le fichier _redirects doit etre a la racine du projet. Tester que les URLs sans .html
fonctionnent bien.
ÔÇó HTTPS : Configurer la redirection HTTP > HTTPS dans Netlify ET dans le fichier _redirects.
ÔÇó JSON-LD : Valider le balisage avec le Rich Results Test de Google apres deploiement.
ÔÇó Images : Toujours au format WebP, avec des noms descriptifs et des attributs alt remplis. Ne
jamais laisser un alt vide.
5.3 Bonnes pratiques de workflow
ÔÇó Un dossier par site : Creer un dossier dedie par projet pour eviter les melanges. Chaque site a
son propre depot GitHub.
ÔÇó Memoire Claude Code : Creer une memoire specifique a chaque projet dans Claude Code pour
maintenir la coherence sur la duree.
ÔÇó Plan de validation : Approuver le plan UNE fois au debut, puis laisser Claude en autonomie.
Eviter les validations intermediaires qui cassent le rythme.
ÔÇó Commit reguliers : Faire un commit a chaque etape significative avec un message clair. Facilite
le rollback en cas de probleme.
ÔÇó Donnees CSV : Pour les donnees brutes (villes, codes postaux, etc.), privilegier le format CSV
comme source de donnees structurees.

5.4 Checklist de lancement
CHECKLIST AVANT MISE EN PRODUCTION
Verifier systematiquement chaque point avant d'envoyer le site au client ou de le declarer en production.
1. Toutes les pages de villes sont generees et accessibles
2. Les Titles sont optimises (mot-cle en debut, pas de separateur)
3. Les meta descriptions sont uniques et orientees CTR
4. Les canonicals pointent vers les bonnes URLs
5. Le maillage interne est coherent et complet
6. Les formulaires Netlify fonctionnent (test d'envoi effectue)
7. Le JSON-LD est valide sur chaque page
8. Le sitemap.xml est complet et soumis a Search Console
9. Les redirections fonctionnent (URLs propres, HTTPS)
10. Le site est responsive (teste sur mobile et desktop)
11. Les images sont en WebP avec alt remplis
12. La vitesse est acceptable (PageSpeed Insights > 90)
13. Les patterns IA sont corriges (accents, tirets, majuscules)
